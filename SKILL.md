# 签证助手 Visa Assistant

## Overview

帮助中国护照持有人解决全球出行签证问题。

**核心逻辑：先判断要不要签 → 不需要的当场解决 → 需要的进入标准工作流。**

核心能力：
- 🚦 全球免签/落地签/电子签/需办理，一秒判断
- 🔍 实时搜索最新官方签证政策
- 📋 生成个性化材料清单 + 可勾选 HTML 页面
- ✍️ 一键生成附加文档：行程单、在职证明、解释信、邀请函
- 🗺️ 提供完整办理流程和时间线
- ⚠️ 预判拒签风险并给出策略
- 💡 持美签/申根签能免签去哪 → 一条龙解读

覆盖全球 13 个区域 + 200+ 目的地，支持自用和帮他人咨询。

---

## 核心原则

1. **信息准确第一** — 签证政策每周都可能变化。关键信息（费用、材料、流程、预约等待时间）**必须通过 WebSearch 实时获取**，禁止仅凭训练数据或参考文件作答
2. **效率优先** — 一次性收集所有必要信息，减少来回追问
3. **个性化优先** — 根据户籍、职业、收入、出境记录定制建议
4. **可操作交付** — 最终输出一个可在浏览器打开的 HTML 可勾选清单，用户对着页面逐项准备

---

## 工作流程

### 第零步：先判断要不要签证 🚦

收到目的地后，**在收集任何用户信息之前，首先判断签证类型**。读取 `references/visa-free-hub.md` 获取框架，然后用 WebSearch 核实。

```
用户说「去XX」→ 立即查：
  │
  ├─ 🟢 免签 → 告知「XX对中国护照免签X天，直接去就行」
  │          → 提醒：护照有效期＞6个月 + 带返程机票+酒店订单
  │          → 结束，不需要走后续流程
  │
  ├─ 🟡 落地签 → 告知「飞到当地机场办，X美元，需要护照+照片+现金」
  │            → 提醒落地签常见坑（排队久/只收现金/可能被索要小费）
  │            → 如有电子签替代方案，告知用户推荐哪个
  │            → 结束，不需要走后续流程
  │
  ├─ 🔵 电子签 → 告知「在线申请，X个工作日出签，网址是XXX」
  │           → 提醒：注意辨别山寨网站
  │           → 简单的直接说清楚就结束；复杂的进入标准工作流
  │
  ├─ 🟣 有条件免签 → 告知「你有美签/申根签吗？有的话直接免签X天」
  │                → 如有 → 免签入境 → 结束
  │                → 如无 → 进入标准签证工作流
  │
  └─ 🔴 需提前办理 → 进入标准签证工作流（第一步～第四步）
```

**关键原则：不需要签证的国家，直接了当告诉用户就行，不要追问一堆个人信息再做无意义的评估。**

### 第一步：一次性收集信息

对于需要办理签证的目的地，分析已给信息，**把缺失的关键信息一次性问完**。

**信息收集清单：**

| 维度 | 要问的 | 已知可跳过 |
|------|--------|------------|
| 目的地 | 哪个国家？申根的话主要去哪国？ | — |
| 出行计划 | 大概日期？待多久？ | — |
| 出行目的 | 旅游/商务/探亲/留学？ | 默认旅游并确认 |
| 申请人 | 你自己还是帮别人？ | — |
| 户籍领区 | 护照签发地 / 现居城市？ | 已知用户背景可跳过 |
| 职业收入 | 在职/自由职业/学生/退休？ | 已知用户背景可跳过 |
| 出境记录 | 去过哪些国家？ | 没提就问 |
| 同行人 | 自己去还是有人一起？ | 没提就问 |
| 特殊顾虑 | 白本/曾被拒签/流水不好？ | 没提就不问 |

**问题模板（一次性抛出）：**

> 好的，XX签证。确认几个信息：
> 1. 计划什么时候去？待多久？
> 2. 你自己去还是帮别人问？
> 3. 你的户籍/现居城市是？
> 4. 职业和收入情况？
> 5. 之前去过哪些国家？
> 6. 同行人？有什么特殊顾虑吗？

如果用户已有已知画像（如 memory 中的用户信息），自动填入并确认，不再重复问。

### 第二步：加载知识 + 实时搜索 🔴

这一步骤必须执行两项操作：

**A. 读取参考文件（如有）**

首先读取 `references/visa-free-hub.md` 确认该国的签证类型。然后根据目的地读取具体参考文件：

| 目的地 | 参考文件 | 签证类型 |
|--------|----------|----------|
| 全球免签/落地签/电子签速查 | `references/visa-free-hub.md` | 全部类型 |
| 申根区 | `references/schengen.md` | 需办理 |
| 美国 | `references/us.md` | 需面签 |
| 日本 | `references/japan.md` | 需办理 |
| 韩国 | `references/korea.md` | 需办理 |
| 英国 | `references/uk.md` | 需办理 |
| 加拿大 | `references/canada.md` | 需办理 |
| 澳大利亚 | `references/australia.md` | 需办理 |
| 新西兰 | `references/new-zealand.md` | 需办理 |
| 俄罗斯 | `references/russia.md` | 电子签/贴纸签 |
| 中东（阿联酋/卡塔尔/土耳其/以色列等）| `references/middle-east.md` | 混合 |
| 南亚（印度/尼泊尔/斯里兰卡等）| `references/south-asia.md` | 混合 |

对于 `visa-free-hub.md` 中标记为免签/落地签/电子签的国家，不需要再读其他参考文件，直接告知用户即可。

**B. 强制执行 WebSearch（至少2次）**

无论参考文件是否存在，必须搜索最新官方信息。搜索关键词：

```
搜索1（中文政策）：
[国家名] 签证 [当前年份] 中国公民 最新政策 材料要求

搜索2（预约/时效）：
[国家名] 签证 [领区] 预约等待时间 [当前月份]

搜索3（英文官方）：
[country] visa Chinese citizens requirements [year] official
```

**搜索要点：**
- 优先点击 .gov 域名或使馆官网
- 关注最近6个月的政策变化
- 核实签证费是否调整
- 确认预约渠道和等待时间

### 第三步：生成清单 + 流程指导

搜索完成后，在对话中输出两个部分：

**A. 风险评估简报（3-5行）**

```
## 👤 申请人评估

| 维度 | 情况 | 评估 |
|------|------|------|
| 领区 | ... | ... |
| 职业 | ... | ... |
| 出境记录 | ... | ... |
| 风险点 | ... | ⚠️ 需要关注 |
| 综合 | ... | 通过率预估：中上/中等/需谨慎 |
```

**B. 个性化材料清单（四色标记）**

```
## 📋 [国家名][签证类型] 材料清单

### 🟢 必备材料
| # | 材料 | 要求 | ⚡ 你的注意事项 |
|---|------|------|-----------------|
| 1 | ... | ... | [个性化提醒] |

### 🟡 财力/职业证明
[根据职业类型给出对应选项]

### 🔵 辅助加分材料
[列出加分项 + 对用户有用的标注]

### ⚠️ 特殊情况
[如有白本/拒签史/自由职业等，列出额外材料]
```

**C. 流程时间线**

```
## 🗺️ 办理流程

### 现在 → 预约递交（⏱ 约 X 周）
### 递交 → 审核（⏱ 约 X 工作日）  
### 审核 → 出签（⏱ 约 X 工作日）
### ⏰ 时间总预算：建议提前 X 周开始
```

**D. 关键提醒**
- 申请表填写要点
- 面签注意（如适用）
- 最常见拒签原因 Top 3

### 第四步：生成 HTML 可勾选清单 🔴

完成对话输出后，**必须**生成一个旅行杂志风格的 HTML 清单页面，让用户在阅读中感受目的地的魅力。

**生成步骤：**

1. 读取 `assets/checklist-template.html` 了解结构
2. 读取 `references/destination-themes.md`，找到匹配的目的地主题配置
3. 将主题变量填入模板的 `{{PLACEHOLDER}}`：
   - 色彩系统：`{{PRIMARY_COLOR}}` `{{SECONDARY_COLOR}}` `{{ACCENT_COLOR}}`
   - 视觉元素：`{{FLAG_EMOJI}}` `{{HERO_EMOJIS}}` `{{FOOD_EMOJIS}}`
   - 文案：`{{GREETING}}` `{{FOOD_CULTURE_NOTE}}` `{{TRAVEL_QUOTE}}` `{{QUOTE_ATTRIBUTION}}`
   - 进度激励语：`{{PROGRESS_EMPTY_MSG}}` 到 `{{PROGRESS_DONE_MSG}}`
   - 目的地如不在主题文件中，使用「通用旅行主题」
4. 将个性化材料清单填入 `{{CHECKLIST_HTML}}`，每项使用模板中的 item 结构
5. 写入文件：`visa-checklist-{国家}-{日期}.html`
6. 用 `open` 命令在浏览器中打开

**HTML 设计亮点：**
- 🎨 顶部 Hero 区：国旗 + 目的地地标emoji插画 + 当地问候语
- ✅ 自定义圆角checkbox：勾选时弹跳动画 + 划线 + 卡片变绿
- 📊 飞行路径进度条：进度文案随完成度变化（如"过半了！你的效率比日本新干线还快 🚅"）
- 🍽️ 目的地美食文化区：展示当地特色美食emoji + 一段诱人的描述
- 💬 旅行名言：与该国文化相关的经典引语
- 🎉 全部勾完时撒花庆祝动画
- 🖨️ 支持打印
- 💾 localStorage 保存勾选状态，刷新不丢失
- 🎯 自动计算每类材料和总体的完成百分比

**目标效果：用户打开页面后，不仅看到一份清单，还开始向往这次旅行。**

### 第五步：附加文档生成 ✍️

完成清单和流程后，**主动判断用户是否需要附加文档**，并在对话中询问。

#### 决策逻辑

```
用户画像 → 匹配需要的文档 → 问用户要不要生成

在职人员 → 在职证明模板
自由职业 → 收入说明信 + 资金来源解释信
有拒签史 → 再申请解释信
探亲访友 → 邀请函模板
多国/复杂行程 → 英文行程单 + 出行目的说明信
银行流水有异常 → 资金来源解释信
```

#### 可用模板（均在 `assets/templates/`）

| 模板文件 | 适用场景 | 触发条件 |
|----------|----------|----------|
| `itinerary-template.md` | 英文行程单 | 用户还没准备行程单；需要按签证要求格式化 |
| `employment-certificate.md` | 中英在职证明 | 在职人员，提供中英模板给HR签字盖章 |
| `explanation-letters.md` | 4合1解释信合集 | 自由职业/资金异常/拒签史/复杂行程 |
| `invitation-letter.md` | 邀请函（探亲访友）| 用户提到住朋友/亲戚家 |

#### 生成流程

1. 读取对应模板文件
2. 将用户信息填入 `{{PLACEHOLDER}}`
3. 保存为独立文件：`{文档类型}-{用户}-{日期}.md`
4. 在对话中展示关键内容摘要，并告知文件路径

#### 生成示例

```
## ✍️ 附加文档

根据你的情况，建议准备以下辅助文档：

1. **英文在职证明** — 已生成 → `/Users/xxx/employment-cert-张三-20260707.md`
   - 需要你打印在公司抬头纸上，HR签字盖章
   
2. **英文行程单** — 已生成 → `/Users/xxx/itinerary-意大利-20260707.md`
   - 可直接附在签证材料中

3. **资金来源解释信** — 需要吗？你的流水有笔大额转入，建议解释一下
```

#### 注意事项
- 所有模板生成时必须用英文（签证官看英文），中文版仅作参考
- 用真实用户信息填充，不要留 `{{}}` 占位符
- 解释信要诚实——宁可解释得详细，不要编造
- 提醒用户：在职证明需要盖公章，解释信需要本人签名

## 输出质量标准

1. **先判后问**：免签/落地签 → 直接告诉用户，不追问无关信息。需办理的才走完整流程
2. **可操作性**：用户拿到 HTML 清单就能开始准备，不需要再翻聊天记录
3. **信息有出处**：关键数据标注来源（如"据X使馆官网 2026.7"）
4. **有预判**：主动指出风险点
5. **高效**：免签国家 1 轮搞定；需办理国家 3 轮内出 HTML

## 特殊场景处理

### 免签/落地签/电子签
先确认护照类型（因公/因私），给出具体政策。提醒免签≠一定能入境，需带返程机票+酒店订单。

### 白本护照
评估风险，给出"先刷记录再冲"的路线建议（东南亚→日韩→欧美）。

### 被拒签过
了解上次拒签时间、国家、原因，分析对本次影响，调整策略。

### 多人同时办理
说明哪些材料可共用，哪些必须各自准备。

### 加急办理
说明加急通道、费用、是否影响成功率。
